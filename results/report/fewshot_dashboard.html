Does diffusion augmentation help few-shot Gaussian Splatting?

Every comparison from the study. Two confounds were found and removed — object-on-black segmentation was crippling the baselines, and the full-320-view ply init was leaking geometry. The honest answer only appears once both are gone. All metrics are object-masked PSNR (higher is better).

The grids

Three findings that reframe the study

The ply leak. The init cloud came from COLMAP on all 320 views, so every "few-shot" run was handed the full object geometry. Worth ~7 PSNR — 20 honest views ≈ 5 views + the full ply.

n20, three inits. Own-ply (COLMAP on the 20 real views) nearly matches the leaky full ply — but few views can't register, so own-ply is n20-only (n10 → 2/10, n5 → 2/5).

The unifying insight — augmentation's value inverts with init quality. With a poor init (honest random, no geometry) synthetic frames add missing supervision and help. With a good init (own-ply ≈ full ply) the geometry is already there, so the same slightly-soft frames only dilute. Inpaint "worked" earlier because the baseline was broken; fix it and little is left to add.

Notes & caveats

SVD hurt is dilution, not a bug. Poses, intrinsics and resolution were verified (object centroid projects onto the tank; 3/4 frames match a model render at their pose). SVD frames are simply soft diffusion output vs sharp real photos — more of them pulls the reconstruction toward that softer average.

Random init is noisy. The n20 r50 cells (inpaint 18.5, svd 15.5) are clear instability outliers — a publishable table needs multiple seeds.

Zero123 (earlier arm, segmented) diverged (~6 PSNR, below baseline); SVD via COLMAP-assigned poses fixed the pose/FOV problem and turned that into neutral/dilution rather than corruption.
